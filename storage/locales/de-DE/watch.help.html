Überwacht Aktionen in Verzeichnissen

Input

Pfad zu einem Verzeichnis/einer Datei oder mehreren Verzeichnissen/Dateien. Werden mehrere Verzeichnisse/Dateien angegeben, so sind diese mit , zu trennen.

Beispiel: /home, /var

Output

payload

Gibt die Ausgeführte Aktion an

file path

Gibt den Pfad der Datei an, auf der eine Aktion ausgeführt wird.

Kann nicht vorhanden sein

path

Gibt den Pfade des Verzeichnisses an, in dem eine Aktion ausgeführt wird

Kann nicht vorhanden sein

type

Gibt den Typ an, auf dem eine Aktion ausgeführt wird. Der Typ kann file oder directory sein

Kann nicht vorhanden sein

Details

Beschreibung des Payloads

  • IN_ACCESS Auf die Datei wurde zugegriffen
  • IN_ATTRIB Metadaten wurden geändert
  • IN_CLOSE_WRITE Die zum Schreiben geöffnete Datei wurde geschlossen
  • IN_CLOSE_NOWRITE Datei oder Verzeichnis, die nicht zum Schreiben geöffnet wurden, wurde geschlossen
  • IN_CREATE Datei oder Verzeichnis im überwachten Verzeichnis erstellt
  • IN_DELETE Datei oder Verzeichnis im überwachten Verzeichnis gelöscht
  • IN_DELETE_SELF Überwachte Datei oder überwachtes Verzeichnis gelöscht
  • IN_MODIFY Datei wurde geändert
  • IN_MOVE_SELF Überwachte Datei oder überwachtes Verzeichnis wurde verschoben
  • IN_MOVE_FROM Wird für das Verzeichnis generiert, das den alten Dateinamen enthält, wenn eine Datei umbenannt wird
  • IN_MOVE_TOWird für das Verzeichnis generiert, das den neuen Dateinamen enthält, wenn eine Datei umbenannt wird
  • IN_OPEN Datei oder Verzeichnis wurde geöffnet
